# 校园工单管理系统前端迭代总结

整理日期：2026-06-24  
项目目录：`campus-work-order-frontend`

## 一、管理员端

### 1. 工单管理 `OrderManage.vue`

- 展示后端新增的 `createdAt` 创建时间。
- 增加开始时间、结束时间查询，参数为 `startTime`、`endTime`。
- 时间格式统一为 `YYYY-MM-DDTHH:mm:ss`。
- 状态下拉查询改为六个标签：待审核、待处理、处理中、已完成、已驳回、已取消。
- 默认显示“待审核”，切换状态后自动回到第一页并重新查询。
- 增加后端分页，支持每页 5、10、20、50 条。
- 查询条件保持单行，窄屏时允许横向滚动。
- 操作列宽度由 320px 缩小为 220px。
- 补齐 `openDetail` 方法和详情弹窗，修复“详情无法查看”的错误。
- 详情展示状态、创建时间、维修人员、问题描述、处理结果和驳回原因。

### 2. 用户管理 `UserManage.vue`

- 接入后端分页接口 `GET /api/users/page`。
- 增加分页参数 `pageNum`、`pageSize` 和总数 `total`。
- 增加 `createdAt` 创建时间列。
- 增加开始时间、结束时间查询。
- 移除 ID 查询框，但管理员列表仍保留 ID 列。
- 查询、重置按钮保持在同一行。
- 状态列移动到列表最后。

## 二、学生端

### 1. 我的工单 `MyOrders.vue`

- 接口：`GET /api/work-orders/my`。
- 请求必须携带 `pageNum`、`pageSize`。
- 使用后端返回的 `list` 和 `total` 完成分页。
- 状态查询改为六个状态标签。
- 增加创建时间范围查询。
- 工单编号最多 20 字符，只允许英文字母和数字，并自动转换成大写。
- 标题最多 50 字符，地点最多 100 字符，并显示字数限制。
- 开始时间晚于结束时间时阻止查询并提示。
- 详情接口：`GET /api/work-orders/{id}`。
- 取消接口：`POST /api/work-orders/{id}/cancel`。

### 2. 新建工单 `CreateOrder.vue`

- 工单标题限制为 2—50 字符。
- 问题描述限制为 5—500 字符。
- 地点限制为 2—100 字符。
- 输入框显示实时字数。
- 禁止只输入空格。
- 提交前去除标题、描述和地点两端空格。

## 三、维修人员端

### 我的任务 `MyTasks.vue`

- 不再区分“当前任务”和“历史任务”。
- 改为三个状态标签：待处理、处理中、已完成。
- 待处理、处理中调用 `GET /api/work-orders/worker/tasks`。
- 已完成调用 `GET /api/work-orders/worker/history`。
- 增加分页、创建时间和时间范围查询。
- 查询栏保持单行。
- 已完成列表显示完成时间。
- 处理结果只在详情弹窗中展示。
- 接单接口：`POST /api/work-orders/{id}/accept`。
- 完成接口：`POST /api/work-orders/{id}/finish`。

## 四、个人信息页面

### `Profile.vue`

- 移除普通用户无实际用途的数据库用户 ID。
- 管理员用户管理列表仍保留 ID，方便管理和排查。
- 用户信息改为双列布局。
- 增加移动端响应式布局。
- 头像仅允许 JPG、PNG、WebP。
- 头像大小不得超过 2MB。
- 增加头像悬停提示、上传状态和失败处理。

## 五、登录与注册页面

- 增加表单校验和防止重复提交。
- 用户名要求非空且不超过 20 字符。
- 密码要求 6—32 字符。

### 3. 注册 `Register.vue`

- 增加确认密码。
- 手机号限制为 11 位有效手机号。
- 增加表单校验和提交状态。
- 用户名要求非空且不超过 20 字符。
- 修复用户名规则过严导致无法注册的问题。
- 捕获表单校验失败，避免 `Uncaught (in promise)`。

## 七、遇到的错误及解决办法

### 1. `AggregateError: ECONNREFUSED`

原因：前端 Vite 代理无法连接 `http://localhost:8080`。  
解决：启动后端，或修改 `vite.config.js` 中的代理 `target`，然后重启前端。

### 2. `_ctx.openDetail is not a function`

原因：模板调用了未定义的 `openDetail`。  
解决：补齐详情方法、详情接口调用和详情弹窗。

### 3. `Name for argument of type Long not specified`

原因：Spring 无法通过反射获取参数名。  
解决：后端显式指定参数名：

```java
@PathVariable("id") Long id
```

如果是查询参数，则使用：

```java
@RequestParam("id") Long id
```

### 4. `MissingServletRequestParameterException: pageNum`

原因：后端接口已改为强制分页，但前端没有传 `pageNum`。  
解决：前端统一携带 `pageNum`、`pageSize`，并处理后端返回的 `list`、`total`。

### 5. 注册时 `Uncaught (in promise)`

原因：用户名规则过严，且 `form.validate()` 校验失败没有被捕获。  
解决：放宽用户名规则，并使用以下方式终止无效提交：

```js
const valid = await formRef.value.validate().catch(() => false)
if (!valid) return
```

## 八、后续建议

- 后端限制 `pageSize` 最大值，建议不超过 50 或 100。
- 前端校验不能代替后端校验，后端仍需验证字段长度、手机号、权限和状态流转。
- 统一前后端时间格式及时区，当前前端使用 `YYYY-MM-DDTHH:mm:ss`。
- 仪表盘建议增加角色范围统计接口，避免使用较大 `pageSize` 拉取列表后在前端统计。
- 在安装 Node.js/npm 的环境执行 `npm run build`。
- 回归测试登录、注册、分页、状态切换、详情、审核、分配、取消、接单和完成流程。
