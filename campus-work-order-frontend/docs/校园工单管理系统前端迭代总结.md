# 校园工单管理系统前端迭代总结

整理日期：2026-06-25  
项目目录：`campus-work-order-frontend/campus-work-order-frontend`

## 一、整体视觉与布局

- 登录、注册页统一使用 `AuthShell.vue`。
- 页面加入安徽师范大学校徽、校名和校园视觉元素。
- 首页仪表盘增加统计卡片、趋势图、类别图、优先级图和角色相关数据区。
- 主系统采用左侧导航、顶部用户区、标签页导航、内容区布局。
- 标签页支持点击切换、关闭当前、关闭其他、关闭全部。
- 表格页面尽量保持单页自适应，内容区域内部滚动。
- 长标题、长地点统一省略展示，悬浮可查看完整内容。
- 刷新按钮统一改为图标按钮。

## 二、登录与注册

### 登录 `Login.vue`

- 支持验证码登录。
- 支持记住账号密码。
- 支持回车登录。
- 增加表单校验和重复提交保护。
- 用户名、密码增加长度限制。

### 注册 `Register.vue`

- 注册页增加身份选择。
- 公开注册只允许选择学生或维修人员。
- 增加确认密码。
- 手机号限制为 11 位手机号。
- 增加表单校验和提交状态。
- 管理员和超级管理员不能通过公开注册创建。

## 三、首页仪表盘 `Dashboard.vue`

- 管理员展示工单总览、状态统计、类别统计、优先级统计和近期工单。
- 学生展示我的工单统计、近期工单、状态概览和处理进度。
- 维修人员展示任务统计、待处理任务、处理历史和完成情况。
- 统计卡片和数据分析区统一展示今日新增，按当前账号可访问工单实时计算。
- 页面加入安徽师范大学视觉风格。
- 数据来源尽量使用真实接口，不使用纯静态假数据。

## 四、管理员端

### 1. 工单管理 `OrderManage.vue`

- 支持工单编号、标题、地点、类别、优先级、状态、创建时间查询。
- 支持待审核、待分配、待处理、处理中、已完成、已驳回、被拒绝、已取消状态切换。
- 默认新工单在前，按创建时间倒序展示。
- 支持分页。
- 支持刷新。
- 待审核支持批量通过。
- 支持审核通过、驳回、分配维修人员。
- 支持查看维修人员拒绝原因，并对被拒绝工单重新分配。
- 维修人员下拉框只展示启用状态的维修人员。
- 工单详情展示基础信息、图片、问题描述、处理结果、驳回原因或拒绝原因。
- 图片支持预览，多图横向滚动展示。
- 标题和地点过长时省略，悬浮显示完整内容。
- 问题描述固定区域展示，内容过多时内部滚动。

### 2. 用户管理 `UserManage.vue`

- 接入用户分页接口 `GET /api/users/page`。
- 左侧增加三级用户分类树。
- 支持用户名、真实姓名、手机号、角色、状态、创建时间查询。
- 表格展示 ID、头像、用户名、真实姓名、手机号、角色、状态和操作。
- 角色、状态使用更紧凑的样式展示。
- 支持新增用户，默认密码为 `123456`。
- 支持查看用户。
- 支持修改真实姓名、手机号、角色、状态。
- 支持单个重置密码。
- 支持批量重置密码。
- 支持逻辑删除用户。
- 支持批量删除用户。
- 删除或禁用维修人员时，其待处理/处理中的工单解除维修人员并退回待处理。
- 维修人员改为学生或管理员时，其待处理/处理中的工单解除维修人员并退回待处理。
- 已完成工单不受维修人员状态、删除、角色变更影响。

## 五、学生端

### 1. 新建工单 `CreateOrder.vue`

- 支持工单标题、类别、优先级、地点、问题描述。
- 支持上传工单图片，上传列表横向滚动展示，删除后不会提交已删除图片地址。
- 标题、地点、描述增加长度校验和空格校验。
- 提交前去除首尾空格。
- 使用 `Idempotency-Key` 防止重复提交。

### 2. 我的工单 `MyOrders.vue`

- 接口：`GET /api/work-orders/my`。
- 支持分页。
- 支持工单编号、标题、地点、类别、优先级、状态、创建时间查询。
- 支持详情查看。
- 支持图片查看，多图横向滚动展示。
- 待审核、待处理、被拒绝工单可取消。
- 待审核、待处理支持批量取消。
- 维修人员也可以访问该页面，用于查看自己曾经提交的工单。

## 六、维修人员端

### 我的任务 `MyTasks.vue`

- 支持待处理、处理中、已完成、已拒绝状态切换。
- 待处理、处理中调用 `GET /api/work-orders/worker/tasks`。
- 已完成调用 `GET /api/work-orders/worker/history`。
- 支持分页、类别、优先级、创建时间查询。
- 待处理支持批量接单。
- 支持接单。
- 支持拒绝接单并填写拒绝原因，已拒绝任务可查看详情。
- 支持完成工单并填写处理结果。
- 支持详情查看和图片预览。
- 维修人员同时保留“我的工单”入口，避免学生转维修人员后看不到历史报修。

## 七、个人信息页面 `Profile.vue`

- 支持修改用户名、真实姓名、手机号。
- 支持单独修改密码。
- 修改密码需要输入原密码、新密码、确认新密码。
- 用户名限制为 3-20 位字母、数字或下划线。
- 真实姓名限制为 2-20 位中文、字母或间隔点。
- 手机号限制为 11 位手机号。
- 密码增加长度和复杂度限制。
- 支持头像上传、预览和背景视觉优化。

## 八、主要接口

| 功能 | 方法与接口 | 说明 |
|---|---|---|
| 登录 | `POST /api/auth/login` | 用户认证 |
| 注册 | `POST /api/auth/register` | 学生/维修人员注册 |
| 当前用户 | `GET /api/auth/me` | 获取当前登录用户 |
| 退出登录 | `POST /api/auth/logout` | 删除 Redis 登录会话 |
| 修改个人信息 | `PUT /api/profile` | 修改用户名、真实姓名、手机号 |
| 修改密码 | `PUT /api/profile/password` | 修改当前用户密码 |
| 上传头像 | `POST /api/files/avatar` | OSS 上传头像 |
| 上传工单图片 | `POST /api/files/work-order-images` | OSS 上传工单图片 |
| 管理员工单分页 | `GET /api/work-orders/page` | 支持分页、条件查询 |
| 我的工单分页 | `GET /api/work-orders/my` | 学生/维修人员查看自己提交的工单 |
| 工单详情 | `GET /api/work-orders/{id}` | 查看完整工单信息 |
| 工单统计 | `GET /api/work-orders/statistics` | 仪表盘统计 |
| 审核通过 | `POST /api/work-orders/{id}/approve` | 管理员操作 |
| 批量审核通过 | `POST /api/work-orders/batch/approve` | 管理员批量操作 |
| 驳回工单 | `POST /api/work-orders/{id}/reject` | 请求体包含 `rejectReason` |
| 分配人员 | `POST /api/work-orders/{id}/assign` | 请求体包含 `handlerId` |
| 取消工单 | `POST /api/work-orders/{id}/cancel` | 提交人取消 |
| 批量取消 | `POST /api/work-orders/batch/cancel` | 提交人批量取消 |
| 维修任务 | `GET /api/work-orders/worker/tasks` | 待处理、处理中 |
| 维修历史 | `GET /api/work-orders/worker/history` | 已完成任务 |
| 接单 | `POST /api/work-orders/{id}/accept` | 维修人员操作 |
| 批量接单 | `POST /api/work-orders/batch/accept` | 维修人员批量操作 |
| 完成工单 | `POST /api/work-orders/{id}/finish` | 请求体包含 `finishResult` |
| 用户分页 | `GET /api/users/page` | 管理端用户查询 |
| 新增用户 | `POST /api/users` | 管理端新增用户 |
| 修改用户 | `PUT /api/users/{id}` | 修改真实姓名、手机号 |
| 修改用户角色 | `PUT /api/users/{id}/role` | 管理端修改角色 |
| 修改用户状态 | `PUT /api/users/{id}/status` | 管理端启用/禁用 |
| 重置密码 | `PUT /api/users/{id}/reset-password` | 重置为 `123456` |
| 批量重置密码 | `PUT /api/users/batch/reset-password` | 批量重置为 `123456` |
| 删除用户 | `DELETE /api/users/{id}` | 逻辑删除 |
| 批量删除用户 | `DELETE /api/users/batch/delete` | 批量逻辑删除 |
| 维修人员列表 | `GET /api/users/workers` | 分配工单下拉框 |

## 九、测试注意事项

- 修改角色后需要退出重新登录，否则 token 中仍可能是旧角色。
- 修改后端权限或接口后，需要重启后端。
- OSS 配置错误会导致头像或工单图片上传失败。
- 删除、禁用、修改维修人员角色后，应回归测试待处理/处理中工单是否退回待处理。
- 注册维修人员后，应确认能看到“我的任务”和“我的工单”。
