# 校园智慧工单系统前端

更新日期：2026-06-25

本模块是校园智慧工单系统前端，基于 Vue 3 + Vite + Element Plus 实现登录注册、首页仪表盘、学生工单、维修任务、管理员工单管理和用户管理。

## 1. 技术栈

- Vue 3
- Vite
- Element Plus
- Axios
- Vue Router
- Pinia

## 2. 页面功能

### 通用功能

- 登录页；
- 注册页；
- 验证码登录；
- 记住账号密码；
- 用户信息页；
- 修改用户名、真实姓名、手机号；
- 修改密码；
- 头像上传与预览；
- 顶部标签页导航，支持关闭当前、关闭其他、关闭全部；
- 安徽师范大学视觉元素；
- 首页仪表盘；
- 路由权限控制；
- Axios 请求封装和 token 自动携带。

### 学生

- 新建工单；
- 上传工单图片；
- 上传区和详情图片支持横向滚动，避免多图撑高页面；
- 我的工单；
- 工单类别查询；
- 工单详情查看；
- 图片预览；
- 待审核/待处理/被拒绝工单取消；
- 批量取消。

### 维修人员

- 我的任务；
- 待处理、处理中、已完成、已拒绝任务切换；
- 批量接单；
- 拒绝接单并填写拒绝原因；
- 完成工单；
- 查看处理历史；
- 可以查看自己曾经提交的“我的工单”。

### 管理员 / 超级管理员

- 工单管理；
- 工单审核、驳回、分配、查看维修人员拒绝详情、重新分配被拒绝工单；
- 批量审核待审核工单；
- 查看工单图片和详情；
- 用户管理；
- 用户分类树；
- 新增用户；
- 修改用户；
- 查看用户；
- 修改角色和状态；
- 重置密码；
- 批量重置密码；
- 删除和批量删除用户。

### 首页仪表盘

- 三端均展示今日新增、状态统计和真实接口数据；
- 学生端展示我的状态概览、近期工单和处理进度；
- 管理员端展示待审核提醒、快捷入口和整体进度；
- 维修人员端展示待处理任务、完成历史和任务进度。

## 3. 安装依赖

进入前端目录：

```bash
cd campus-work-order-frontend/campus-work-order-frontend
npm install
```

## 4. 启动开发环境

```bash
npm run dev
```

访问：

```text
http://localhost:5173
```

## 5. 构建

```bash
npm run build
```

构建产物位于：

```text
dist
```

## 6. 后端代理

开发环境通过 Vite 代理访问后端：

```text
/api -> http://localhost:8080
```

请先启动 Spring Boot 后端。

## 7. Docker / Nginx 运行

前端 Dockerfile 使用 Node 构建静态资源，再通过 Nginx 提供访问服务。

项目根目录的 `docker-compose.yml` 会同时启动前端、后端、MySQL、Redis 和 RabbitMQ：

```bash
cd ../..
cp .env.example .env
docker compose up -d --build
```

容器运行时访问：

```text
http://localhost
```

Nginx 已配置：

```text
/api -> http://backend:8080
```

因此容器环境下前端不需要额外配置后端地址。

## 9. 权限说明

- 学生可以新建工单和查看自己的工单；
- 维修人员可以处理任务，也可以查看自己以前提交的工单；
- 管理员可以管理工单和普通用户；
- 超级管理员可以管理管理员、维修人员和学生；
- 管理员和超级管理员不能通过公开注册产生，只能由已有管理账号新增。

## 10. 开发注意事项

- 修改权限相关逻辑后，测试账号需要退出重新登录，因为 JWT 中包含角色信息；
- 上传图片依赖后端 OSS 配置；
- 表格和详情页对长标题、长地点做了省略和悬浮查看；
- 单页布局尽量控制在当前视口内，表格区域内部滚动；
- 前端校验只负责体验，后端仍会做权限、状态和字段校验。
