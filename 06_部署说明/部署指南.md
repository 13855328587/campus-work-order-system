## 1. 文档说明

本文档用于说明校园工单管理系统的本地运行、Docker Compose 部署、CI/CD 流程和常用运维命令，便于项目验收、复现和后续维护。

项目支持两种运行方式：

- 本地开发运行：前端、后端、数据库等服务分别启动，适合开发调试。
- Docker Compose 一键部署：一次性启动前端、后端、MySQL、Redis、RabbitMQ，适合演示和部署验证。

## 2. 项目组成

| 模块 | 技术栈 | 路径 |
|---|---|---|
| 后端服务 | Spring Boot 3、Spring Security、MyBatis Plus | `campus-work-order-backend/` |
| 前端页面 | Vue 3、Vite、Element Plus、Pinia | `campus-work-order-frontend/campus-work-order-frontend/` |
| 数据库 | MySQL 8 | Docker Compose 或本地 MySQL |
| 缓存 | Redis | Docker Compose 或本地 Redis |
| 消息队列 | RabbitMQ | Docker Compose |
| 文件上传 | 阿里云 OSS | 通过环境变量配置 |
| CI/CD | GitHub Actions | `.github/workflows/ci.yml` |
| 性能测试 | k6 | `performance/` |

## 4. 环境变量配置

项目根目录提供环境变量模板：

```text
.env.example
```

部署前复制为：

```bash
cp .env.example .env
```

Windows PowerShell 可手动复制：

```powershell
Copy-Item .env.example .env
```

常见配置项：

```text
MYSQL_ROOT_PASSWORD=数据库 root 密码
MYSQL_DATABASE=campus_work_order
JWT_SECRET=JWT 签名密钥
REDIS_DATABASE=0
RABBITMQ_USERNAME=RabbitMQ 用户名
RABBITMQ_PASSWORD=RabbitMQ 密码
OSS_ENDPOINT=阿里云 OSS Endpoint
OSS_BUCKET_NAME=OSS Bucket 名称
OSS_ACCESS_KEY_ID=OSS AccessKey ID
OSS_ACCESS_KEY_SECRET=OSS AccessKey Secret
OSS_URL_PREFIX=OSS 文件公网访问前缀
```

注意：

- `.env` 是本地真实配置文件，不应提交到 GitHub。
- `.env.example` 只保留示例值，可以提交。
- OSS 密钥、数据库密码、JWT 密钥不要写入公开仓库。

## 5. Docker Compose 一键部署

在项目根目录执行：

```bash
docker compose up -d --build
```

启动后访问：

```text
前端系统：http://localhost
后端接口：http://localhost:8080
RabbitMQ 管理页：http://localhost:15672
```

Docker Compose 会启动以下服务：

| 服务 | 说明 |
|---|---|
| frontend | Vue 前端页面，由 Nginx 托管 |
| backend | Spring Boot 后端接口服务 |
| mysql | 业务数据库 |
| redis | 缓存服务 |
| rabbitmq | 消息队列服务 |

## 6. 数据库初始化

初始化 SQL 文件位置：

```text
campus-work-order-backend/src/main/resources/sql/init.sql
```

首次创建 MySQL 数据卷时，Docker 会自动执行该 SQL 文件。

如果需要清空 Docker 环境数据库并重新初始化：

```bash
docker compose down -v --remove-orphans
docker compose up -d --build
```

注意：`docker compose down -v` 会删除 Docker 数据卷，会清空 Docker 环境中的数据库数据。

当前初始化数据包含：

- 超级管理员、管理员、维修人员、学生测试账号
- 待审核工单
- 待分配工单
- 待处理工单
- 处理中工单
- 已完成工单
- 被拒绝工单
- 已取消工单
- 操作日志与幂等记录相关表结构

## 7. 本地开发运行

### 7.1 启动后端

进入后端目录：

```bash
cd campus-work-order-backend
```

启动 Spring Boot：

```bash
mvn spring-boot:run
```

后端默认端口：

```text
http://localhost:8080
```

进入前端目录：

```bash
cd campus-work-order-frontend/campus-work-order-frontend
```

```bash
npm install
```

```bash
npm run dev
```

```text
http://localhost:5173
```

## 9. 常用运维命令

查看容器状态：

```bash
docker compose ps
```

查看日志：

```bash
docker compose logs -f backend
docker compose logs -f frontend
```

停止服务但保留数据：

```bash
docker compose down
```

重新构建并启动：

```bash
docker compose up -d --build
```

清空数据并重新初始化：

```bash
docker compose down -v --remove-orphans
docker compose up -d --build
```

## 11. 部署注意事项

- 修改后端代码后，需要重新启动后端或重新构建 Docker 镜像。
- 修改前端代码后，生产环境需要重新执行前端构建。
- 修改初始化 SQL 后，如果已有 Docker 数据卷，必须执行 `docker compose down -v` 才会重新初始化。
- 修改用户角色或权限后，建议退出重新登录，因为 JWT 中可能包含登录时的角色信息。
- 删除、禁用或改变维修人员角色时，系统会将其未完成工单退回待分配，避免工单无法继续处理。
- 如果图片无法访问，应检查 OSS Bucket 权限、URL 前缀和后端 OSS 配置。
- 真实密码、OSS 密钥、Token 不应提交到 GitHub。

## 12. 验收说明

部署完成后，可按以下顺序检查：

1. 访问 `http://localhost`，确认前端页面正常打开。
2. 使用默认账号登录系统。
3. 学生端新建工单并上传图片。
4. 管理员端审核、分配工单。
5. 维修人员端接单、拒单或完成工单。
6. 管理员端查看状态流转和仪表盘统计。
7. 检查 RabbitMQ、Redis、MySQL 容器是否正常运行。

如果以上流程均正常，说明系统部署成功。
